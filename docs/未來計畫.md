# 未來計畫：AI 導航站 + 部落格整合方案

> 記錄時間：2025.12.01  
> 目標：建立一個整合 AI 工具導航與個人部落格的站點

## 📋 需求分析

### 核心需求
1. **AI 主題導航站/目錄頁**：收錄 AI 工具、資源、課程等連結
2. **個人部落格文章**：發布學習筆記、技術文章

### 期望效果
- 一個域名，兩種功能
- 兩者可以互相連結
- 便於維護和更新

---

## 🎯 採用方案：NotionNext 站點合併

### 技術方案
使用 NotionNext 的「站點合併」功能（Multi-Site Combine）

**參考文件**：
- [多語言站點](https://docs.tangly1024.com/article/notion-next-mulity-languages)
- [站點合併](https://docs.tangly1024.com/article/notion-next-site-combine)

### 架構設計

```
一個 Vercel 部署
    ├── 資料庫 A：AI 導航/目錄
    │   ├── 主題：Nav
    │   ├── 路徑：yoursite.com/nav/...
    │   └── 內容：AI 工具、資源、課程連結
    │
    └── 資料庫 B：部落格文章
        ├── 主題：Next (目前使用)
        ├── 路徑：yoursite.com/ 或 yoursite.com/blog/...
        └── 內容：個人文章、學習筆記
```

---

## ⚙️ 設定方式

### 1. blog.config.js 設定

```javascript
// 兩個資料庫合併，用逗號分隔
NOTION_PAGE_ID: '資料庫B的ID,nav:資料庫A的ID'
// 或
NOTION_PAGE_ID: 'nav:資料庫A的ID,blog:資料庫B的ID'

// 第一個資料庫會是預設首頁
```

### 2. Notion 資料庫結構

#### 資料庫 A (導航站)
- `type = Page`：導航卡片
- `category`：分類 (如：AI 工具、AI 課程、AI 資源)
- `URL`：外部連結
- `封面圖`：工具/資源的截圖
- `summary`：簡介

#### 資料庫 B (部落格)
- `type = Post`：部落格文章
- 維持現有結構

---

## 🔗 互聯方式

### 1. 導航頁指向文章
在導航資料庫的條目中，填入內部連結：
```
/article/your-post-slug
```

### 2. 文章指向導航
在文章中提到某個工具時，連結到：
```
/nav/tool-name
```

### 3. 頂部選單互聯
在 Notion 建立 `type = Menu` 的頁面，設定跨資料庫連結

---

## ✅ 方案優勢

1. **資料分離清晰**  
   導航和文章不會混在一起，各自有獨立的資料庫結構

2. **維護方便**  
   兩個資料庫可以有不同的欄位和組織方式

3. **擴充性強**  
   未來可以再加第三個資料庫（如：作品集、工作坊）

4. **SEO 統一**  
   所有內容都在同一個域名下，有利於搜尋引擎優化

5. **一次部署**  
   只需要一個 Vercel 專案，不用維護多個站點

---

## ⚠️ 注意事項

### 權限設定
- 確認兩個 Notion 資料庫都設為「公開」
- 或都綁定同一個 Notion 帳號

### 路徑規劃
- 路徑前綴（nav, blog）要在一開始就決定好
- 一旦上線後改路徑，舊連結會失效

### 主題選擇
- 資料庫 A 建議用 **Nav 主題**（專為導航設計）
- 資料庫 B 維持 **Next 主題**（目前使用中）
